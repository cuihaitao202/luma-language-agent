# Luma submission package

## Track

Education

## Tagline

Don’t study a language. Live it.

## Short description

Luma is a personal second-language agent for busy adults. It turns the learner's next real-life moment into a three-minute hear → say → refine → reappear loop, then brings useful language back in a new context before it is forgotten.

## Full description

Adults rarely fail at languages because they lack content. They fail because courses are detached from their lives, speaking feels risky, and isolated vocabulary disappears before it becomes usable.

Luma replaces the syllabus with the learner's next meaningful act: order coffee before a meeting, clarify a point at work, or understand an airport change. The learner listens for meaning, says what they genuinely need, receives one high-value refinement from GPT-5.6, and sees that expression reappear later in a different context. This trains retrieval and transfer without asking the learner to maintain word lists or find a language environment.

The prototype supports eight target languages, adaptive onboarding, browser voice input and output, GPT-5.6 structured feedback, a judge-safe fallback, and a living memory map. Personal preferences stay on the device; the OpenAI key stays server-side.

Codex took Luma from an empty workspace to a tested and deployed product during Build Week. It helped turn the learning thesis into interaction decisions, implemented the responsive frontend and Cloudflare worker, integrated the Responses API, generated the share experience, tested the end-to-end path, and prepared the submission.

## 2:15 demo narration

**0:00–0:18 — Problem**

“Most adults don't need another language course. They need a patient teacher inside the life they already have. Luma turns the next real moment in your day into a lesson your brain can use.”

**0:18–0:35 — Personalization**

“I choose one of eight languages, tell Luma what I need language for, and choose a three-minute rhythm. There is no placement test; Luma adapts from what I understand and say.”

**0:35–1:18 — Working loop**

“My meeting starts soon, so Luma prepares a coffee-shop moment. First I listen for meaning instead of translating every word. Then I say what I would actually order. GPT-5.6 Terra receives the transcript through a server-side Responses API call. It recognizes that I communicated successfully and chooses only one high-value refinement, so correction never overwhelms the learner.”

**1:18–1:40 — Memory transfer**

“Luma doesn't store a vocabulary list. It stores a useful connection between intention, situation, phrase, and successful use. Tomorrow, ‘Could I get…’ returns in a meeting as a request for clarification. That change of context is the point: usable transfer, not short-term memorization.”

**1:40–2:05 — Codex and implementation**

“I built Luma with Codex during OpenAI Build Week. Codex helped turn the adult-learning problem into this product mechanism, implemented the responsive experience and secure Cloudflare worker, integrated GPT-5.6 structured outputs, and ran the build, packaging, and end-to-end validation loop.”

**2:05–2:15 — Close**

“Luma makes the learner's life the curriculum: hear it, say it, refine one thing, and meet it again before it disappears.”
